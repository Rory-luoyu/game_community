# Swagger API文档测试指南

## 测试环境准备
1. 确保后端服务器运行中
2. 确保Swagger依赖已正确添加到pom.xml
3. 确保SwaggerConfig配置类已正确配置
4. 确保WebConfig中已添加Swagger静态资源路径

## 测试步骤

### 1. 访问Swagger UI界面

#### 1.1 基本访问测试
**测试步骤**：
1. 启动后端服务器
2. 在浏览器中访问：`http://localhost:10241/swagger-ui/`
3. 观察页面是否正常显示

**预期结果**：
- 显示Swagger UI界面
- 页面标题为"游戏社区API文档"
- 显示API的基本信息

#### 1.2 API文档内容测试
**测试步骤**：
1. 在Swagger UI页面中查看API列表
2. 检查是否显示所有Controller中的接口
3. 展开各个接口查看详细信息

**预期结果**：
- 显示所有Controller中的接口
- 每个接口显示请求方法（GET、POST等）
- 每个接口显示请求路径
- 每个接口显示请求参数和响应格式

### 2. API在线测试

#### 2.1 GET请求测试
**测试步骤**：
1. 在Swagger UI中找到一个GET请求接口（如获取帖子列表）
2. 点击"Try it out"按钮
3. 填写必要的参数
4. 点击"Execute"按钮执行请求
5. 查看响应结果

**预期结果**：
- 请求成功执行
- 显示请求URL
- 显示响应状态码
- 显示响应体内容

#### 2.2 POST请求测试
**测试步骤**：
1. 在Swagger UI中找到一个POST请求接口（如用户登录）
2. 点击"Try it out"按钮
3. 在Request body中填写JSON格式的请求数据
4. 点击"Execute"按钮执行请求
5. 查看响应结果

**预期结果**：
- 请求成功执行
- 显示请求URL
- 显示请求体
- 显示响应状态码
- 显示响应体内容

#### 2.3 带认证的请求测试
**测试步骤**：
1. 先通过登录接口获取登录信息
2. 在需要认证的接口中添加认证信息
3. 执行请求
4. 查看响应结果

**预期结果**：
- 认证信息正确传递
- 请求成功执行
- 返回正确的响应数据

### 3. API文档验证

#### 3.1 接口信息完整性测试
**测试步骤**：
1. 检查每个接口是否显示完整的请求信息
2. 检查参数说明是否清晰
3. 检查响应格式是否正确

**预期结果**：
- 每个接口显示完整的请求信息
- 参数说明清晰明确
- 响应格式正确显示

#### 3.2 响应格式验证测试
**测试步骤**：
1. 执行多个接口请求
2. 检查响应格式是否统一
3. 验证RtnData格式的正确性

**预期结果**：
- 所有接口返回统一的RtnData格式
- 包含flag、msg、data字段
- 成功和失败的响应格式正确

### 4. 错误处理测试

#### 4.1 错误请求测试
**测试步骤**：
1. 发送错误的请求参数
2. 发送缺少必需参数的请求
3. 查看错误响应

**预期结果**：
- 显示清晰的错误信息
- 返回统一的错误响应格式
- 错误信息有助于调试

#### 4.2 异常处理测试
**测试步骤**：
1. 触发后端异常（如数据库连接失败）
2. 在Swagger UI中查看响应
3. 验证错误信息是否正确显示

**预期结果**：
- 显示全局异常处理器返回的错误信息
- 错误信息格式统一
- 错误信息有助于问题排查

## 常见问题解决

### 问题1：访问Swagger UI显示404
**解决方案**：
- 检查WebConfig中是否正确配置了Swagger静态资源路径
- 检查Swagger依赖是否正确添加到pom.xml
- 检查SwaggerConfig配置类是否正确配置

### 问题2：API文档中没有显示接口
**解决方案**：
- 检查SwaggerConfig中的basePackage是否正确设置为"com.controller"
- 检查Controller类是否使用了@RestController注解
- 检查Controller类是否在正确的包路径下

### 问题3：执行请求时出现错误
**解决方案**：
- 检查后端服务器是否正常运行
- 检查数据库连接是否正常
- 检查请求参数格式是否正确
- 查看后端日志文件获取详细错误信息
