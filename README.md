# game_community
基于SpringBoot+Vue的前后端分离项目，仿照米游社实现基本论坛功能

## 项目功能展示
[![ppxgqZ4.png](https://s1.ax1x.com/2023/04/13/ppxgqZ4.png)](https://imgse.com/i/ppxgqZ4)

## 项目后端

### 后端依赖
1. jdk 11.0.4
2. mysql 8.0
3. redis

### 后端起步
1. 按照提示修改application.yml和application-open.yml文件
2. 启动Application启动类

### 后端配置说明

#### 日志配置
- 日志文件位置：`user.home/logs/game_community/`
- 生成文件：`info.log` 和 `error.log`
- 日志格式：`yyyy-MM-dd HH:mm:ss.SSS [thread] level logger - message`
- 日志滚动：文件超过10MB自动滚动，最多保留30天

#### API文档访问
启动后端服务器后，访问：`http://localhost:10241/swagger-ui/`

## 项目前端

### 前端依赖

## 测试指南

### 测试环境准备
1. 确保前端开发服务器运行中
2. 确保后端服务器运行中
3. 确保数据库连接正常
4. 确保Redis服务运行中（如果使用缓存）
5. 打开浏览器开发者工具（F12）查看网络请求和错误信息

### Swagger API文档测试

#### 访问Swagger UI界面
1. 启动后端服务器
2. 在浏览器中访问：`http://localhost:10241/swagger-ui/`
3. 页面标题为"游戏社区API文档"

#### API在线测试
- **GET请求测试**：点击"Try it out"按钮，填写参数，点击"Execute"执行请求
- **POST请求测试**：在Request body中填写JSON格式的请求数据，执行请求
- **带认证的请求测试**：先通过登录接口获取登录信息，再在需要认证的接口中添加认证信息

#### API文档验证
- 检查每个接口是否显示完整的请求信息
- 检查参数说明是否清晰
- 验证RtnData格式的正确性（包含flag、msg、data字段）

#### Swagger常见问题解决

| 问题 | 解决方案 |
|------|----------|
| 访问Swagger UI显示404 | 检查WebConfig中Swagger静态资源路径配置、Swagger依赖、SwaggerConfig配置 |
| API文档中没有显示接口 | 检查SwaggerConfig的basePackage设置、Controller的@RestController注解 |
| 执行请求时出现错误 | 检查后端服务器、数据库连接、请求参数格式，查看后端日志 |

### 前端错误处理测试

#### 测试场景

| 场景 | 测试步骤 | 预期结果 |
|------|----------|----------|
| 网络错误 | 停止后端服务器，访问需要API的页面 | 显示"网络错误，请稍后重试" |
| 401未授权 | 清除sessionStorage登录信息，访问需要登录的API | 显示"未授权，请重新登录" |
| 403拒绝访问 | 普通用户访问管理员接口 | 显示"拒绝访问" |
| 404页面不存在 | 请求不存在的路径 | 显示"请求地址不存在" |
| 500服务器错误 | 触发后端异常 | 显示"服务器内部错误"及具体信息 |
| 业务逻辑错误 | 重复注册、错误验证码等 | 显示业务相关的错误提示 |

#### 前端错误处理检查清单
- [ ] 网络错误时显示正确提示
- [ ] 401错误时提示重新登录
- [ ] 403错误时提示拒绝访问
- [ ] 404错误时提示地址不存在
- [ ] 500错误时提示服务器错误
- [ ] 业务错误时显示具体错误信息
- [ ] 错误提示使用Element UI的Message组件

### 后端全局异常处理测试

#### 全局异常处理测试

| 异常类型 | 测试步骤 | 预期结果 |
|----------|----------|----------|
| 空指针异常 | 在Controller中添加空指针异常代码 | 返回统一错误响应格式，日志记录异常堆栈 |
| 数据库异常 | 停止数据库服务，访问需要数据库的接口 | 返回统一错误响应格式，日志记录连接异常 |
| 参数验证异常 | 发送缺少必需参数的请求 | 返回参数验证失败错误信息 |

#### 日志记录测试

| 测试项 | 预期结果 |
|--------|----------|
| 日志文件生成 | 在`user.home/logs/game_community/`目录下生成info.log和error.log |
| 日志内容 | INFO级别记录到info.log，ERROR级别记录到error.log |
| 日志滚动 | 文件超过10MB自动创建新文件，最多保留30天 |

#### RtnData统一响应格式

**成功响应示例**：
```json
{
  "flag": true,
  "msg": "操作成功",
  "data": {...}
}
```

**失败响应示例**：
```json
{
  "flag": false,
  "msg": "登录失败",
  "data": null
}
```
