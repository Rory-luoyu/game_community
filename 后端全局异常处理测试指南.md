# 后端全局异常处理测试指南

## 测试环境准备
1. 确保后端服务器运行中
2. 确保数据库连接正常
3. 确保Redis服务运行中（如果使用缓存）
4. 查看日志文件位置：`user.home/logs/game_community/`

## 测试场景

### 1. 全局异常处理测试

#### 1.1 空指针异常测试
**测试步骤**：
1. 在任意Controller方法中临时添加空指针异常代码
2. 访问该接口
3. 检查返回的响应和日志

**预期结果**：
- 返回统一的错误响应格式：`{"flag": false, "msg": "服务器内部错误: ...", "data": null}`
- 日志文件中记录详细的异常堆栈信息
- 请求路径被记录在日志中

#### 1.2 数据库异常测试
**测试步骤**：
1. 停止数据库服务
2. 访问需要数据库操作的接口
3. 检查返回的响应和日志

**预期结果**：
- 返回统一的错误响应格式
- 日志中记录数据库连接异常

#### 1.3 参数验证异常测试
**测试步骤**：
1. 发送缺少必需参数的请求
2. 检查返回的响应和日志

**预期结果**：
- 返回参数验证失败的错误信息
- 日志中记录验证异常

### 2. 日志记录测试

#### 2.1 日志文件生成测试
**测试步骤**：
1. 启动后端服务器
2. 执行一些操作（登录、发帖等）
3. 检查日志文件是否生成

**预期结果**：
- 在`user.home/logs/game_community/`目录下生成日志文件
- 生成`info.log`和`error.log`文件

#### 2.2 日志内容测试
**测试步骤**：
1. 查看info.log文件内容
2. 查看error.log文件内容
3. 验证日志格式是否正确

**预期结果**：
- 日志格式：`yyyy-MM-dd HH:mm:ss.SSS [thread] level logger - message`
- INFO级别日志记录到info.log
- ERROR级别日志记录到error.log
- 日志包含请求路径、异常堆栈等信息

#### 2.3 日志滚动测试
**测试步骤**：
1. 生成大量日志（可以通过循环调用接口）
2. 检查是否生成滚动日志文件

**预期结果**：
- 当日志文件超过10MB时，自动创建新的日志文件
- 日志文件命名格式：`info-2026-03-09.0.log`
- 最多保留30天的日志文件

### 3. RtnData统一响应格式测试

#### 3.1 成功响应测试
**测试步骤**：
1. 调用成功的接口（如获取帖子列表）
2. 检查响应格式

**预期结果**：
```json
{
  "flag": true,
  "msg": "操作成功",
  "data": {...}
}
```

#### 3.2 失败响应测试
**测试步骤**：
1. 调用失败的接口（如使用错误密码登录）
2. 检查响应格式

**预期结果**：
```json
{
  "flag": false,
  "msg": "登录失败",
  "data": null
}
```

#### 3.3 带数据的成功响应测试
**测试步骤**：
1. 调用返回数据的接口（如获取用户信息）
2. 检查响应格式

**预期结果**：
```json
{
  "flag": true,
  "msg": "操作成功",
  "data": {
    "userId": 1,
    "userName": "test",
    ...
  }
}
```
