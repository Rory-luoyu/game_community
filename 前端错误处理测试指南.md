# 前端错误处理测试指南

## 测试环境准备
1. 确保前端开发服务器运行中
2. 确保后端服务器运行中
3. 打开浏览器开发者工具（F12）查看网络请求和错误信息

## 测试场景

### 1. 网络错误测试
**测试步骤**：
1. 停止后端服务器
2. 尝试访问需要API的页面（如首页、帖子详情等）
3. 观察是否显示"网络错误，请稍后重试"的错误提示

**预期结果**：
- 显示友好的错误提示
- 控制台记录错误日志

### 2. 401未授权错误测试
**测试步骤**：
1. 清除sessionStorage中的登录信息
2. 尝试访问需要登录的API（如发布帖子、评论等）
3. 观察错误提示

**预期结果**：
- 显示"未授权，请重新登录"的错误提示
- 可能触发登录对话框

### 3. 403拒绝访问测试
**测试步骤**：
1. 使用普通用户账号登录
2. 尝试访问管理员接口（如果存在）
3. 观察错误提示

**预期结果**：
- 显示"拒绝访问"的错误提示

### 4. 404页面不存在测试
**测试步骤**：
1. 修改API请求地址为不存在的路径
2. 观察错误提示

**预期结果**：
- 显示"请求地址不存在"的错误提示

### 5. 500服务器内部错误测试
**测试步骤**：
1. 在后端代码中临时添加一个异常（如除零错误）
2. 访问相关接口
3. 观察错误提示

**预期结果**：
- 显示"服务器内部错误"的错误提示
- 显示具体的错误信息

### 6. 业务逻辑错误测试
**测试步骤**：
1. 尝试重复注册已存在的邮箱
2. 使用错误的验证码登录
3. 尝试删除不存在的帖子

**预期结果**：
- 显示业务逻辑相关的错误提示
- 提示信息清晰明确

## 测试检查清单
- [ ] 网络错误时显示正确提示
- [ ] 401错误时提示重新登录
- [ ] 403错误时提示拒绝访问
- [ ] 404错误时提示地址不存在
- [ ] 500错误时提示服务器错误
- [ ] 业务错误时显示具体错误信息
- [ ] 错误提示使用Element UI的Message组件
- [ ] 控制台正确记录错误日志
